# Log Action: $ARGUMENTS

Append a structured action entry to today's action log file at `docs/logs/actions_<YYYY-MM-DD>.md`.

## Steps

1. **Determine today's date** and the log file path:
   ```
   docs/logs/actions_YYYY-MM-DD.md
   ```

2. **Create the file** if it does not exist, with this header:
   ```md
   # Action Log — YYYY-MM-DD
   _Auto-generated by Claude Code. Do not edit manually during an active session._
   ```

3. **Append the entry** using this format:
   ```md
   ## [HH:MM] $ARGUMENTS

   **Type**: create | modify | delete | run | migrate | refactor
   **Files affected**:
   - `path/to/file` — description of change
   **Command run** _(if applicable)_: `command here`
   **Reason**: Why this step was taken
   **Rollback**: Exact steps to undo (e.g., "Delete `path/to/file`" or "Run `dotnet ef migrations remove -p Persistence -s API`")
   ```

4. **Rollback field must be specific** — not "revert the change" but the exact command or action needed.

## When to Use

Call `/log-action <title>` before or after every meaningful step:
- Creating or deleting a file
- Running a migration
- Running a build or test command
- Making a structural change (adding a new handler, controller endpoint, component)
- Any destructive operation

## Rollback Reference

| Action type | Rollback |
|---|---|
| New file created | `rm path/to/file` or `git checkout HEAD -- path/to/file` |
| File modified | `git checkout HEAD -- path/to/file` |
| Migration added | `dotnet ef migrations remove -p Persistence -s API` |
| npm package added | `npm uninstall <package> --prefix client` |
| NuGet package added | `dotnet remove <project> package <PackageName>` |
| Database seeded | Restore from backup or re-run DbInitializer with clean DB |

## Session Summary

At the end of a session, append:
```md
## Session Summary

**Steps completed**: N
**Files created**: list
**Files modified**: list
**Files deleted**: list
**Migrations added**: list (or none)
**Tests status**: X passed / Y failed / Z skipped
```
